**Input**: Feature specification from `/specs/005-wave-step-labels/spec.md`

## Summary

Three frontend-only changes to the wave card progress indicators:
1. Remove the 3px top border override on `.wave-card--active` — make all four borders 1.5px.
2. Restructure each `.wave-step` span from a flat circle to three sub-elements: checkbox indicator, status emoji slot, field name label.
3. Update `updateWaveProgress()` in `app.js` to target the new sub-elements instead of setting `textContent` on the outer span.

No backend changes. No new dependencies.

## Architecture Decisions

### HTML: Step sub-element structure

Each `.wave-step` becomes:
```html
<span class="wave-step" data-wave="basic" data-field="target" aria-label="Target: empty">
  <span class="wave-step__check" aria-hidden="true"></span>
  <span class="wave-step__status" aria-hidden="true"></span>
  <span class="wave-step__label">Target</span>
</span>
```

The field name is static HTML. `aria-hidden="true"` on `__check` and `__status` keeps the screen reader experience clean — the outer `aria-label` carries the full description.

### CSS: Wave progress layout change

`.wave-progress` changes from a horizontal row to a vertical column:
- `flex-direction: column` (was default `row`)
- `gap: var(--space-1)` (tighter than the old `var(--space-2)`)

Each `.wave-step` becomes a flex row: `[12px circle] [1.2em status slot] [label text]`

The `.wave-step--filled` class moves its styling target from the outer `.wave-step` to `.wave-step--filled .wave-step__check` (child selector). The outer class remains for CSS cascade convenience.

### JS: Targeted sub-element updates

In `updateWaveProgress()`, after querying the outer `.wave-step`:
1. Query `.wave-step__check` and `.wave-step__status` as children
2. Toggle `.wave-step--filled` on the outer step (unchanged — class cascade drives CSS)
3. Set `statusEl.textContent` to the emoji string or `''`
4. Update `aria-label` on outer step (unchanged logic)
5. Do NOT set `step.textContent` (would destroy child elements)
