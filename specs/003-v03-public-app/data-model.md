# Data Model: Lean Storytelling v0.3

**Branch**: `003-v03-public-app`
**Date**: 2026-03-27

## Entities

### User

Represents a registered account. Created automatically on first magic link login.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| id | UUID | PK, NOT NULL | Generated by DB |
| email | VARCHAR(255) | UNIQUE, NOT NULL | Lowercased on write |
| job_title | VARCHAR(100) | NULL | Optional profile field |
| intent | VARCHAR(200) | NULL | Optional profile field |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | |
| onboarded_at | TIMESTAMPTZ | NULL | Set when profile modal is dismissed |

**Indexes**: `email` (unique)

---

### Story

A named container for a user's work. Has one or more versions.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| id | UUID | PK, NOT NULL | Generated by DB |
| user_id | UUID | FK → users.id ON DELETE CASCADE, NOT NULL | |
| title | VARCHAR(100) | NOT NULL | Min 1 char |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | |

**Indexes**: `user_id`

**Cascade**: Deleting a User deletes all their Stories. Deleting a Story deletes all its StoryVersions.

---

### StoryVersion

A snapshot of a Story at a point in time. The latest version is editable; all others are read-only.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| id | UUID | PK, NOT NULL | Generated by DB |
| story_id | UUID | FK → stories.id ON DELETE CASCADE, NOT NULL | |
| version_number | INTEGER | NOT NULL | Starts at 1; auto-incremented per story |
| context | TEXT | NULL | Wave 3 field |
| target | TEXT | NULL | Wave 1 field |
| empathy | TEXT | NULL | Wave 2 field |
| problem | TEXT | NULL | Wave 1 field |
| consequences | TEXT | NULL | Wave 2 field |
| solution | TEXT | NULL | Wave 1 field |
| benefits | TEXT | NULL | Wave 2 field |
| why | TEXT | NULL | Wave 3 field |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | Set on version creation, never updated |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | Updated on every auto-save PATCH |

**Indexes**: `story_id`, `(story_id, version_number)` (unique)

**Constraint**: `(story_id, version_number)` must be unique — enforced at DB level.

**Latest version**: Determined by `MAX(version_number)` per `story_id`. No `is_latest` flag (computed on read to avoid sync issues).

---

### MagicLinkToken

A short-lived, single-use credential for passwordless authentication.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| id | UUID | PK, NOT NULL | Generated by DB |
| email | VARCHAR(255) | NOT NULL | Lowercased on write |
| token_hash | VARCHAR(64) | NOT NULL | SHA-256 hex of the raw token |
| expires_at | TIMESTAMPTZ | NOT NULL | created_at + 15 minutes |
| used_at | TIMESTAMPTZ | NULL | Set on first use; token is invalid after this |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT now() | |

**Indexes**: `token_hash` (for fast lookup on verify), `email + created_at` (for rate limiting)

**Cleanup**: Rows where `created_at < now() - interval '24 hours'` can be deleted by a background job.

## State Transitions

### StoryVersion — Editability

```
version_number < MAX(version_number) for this story
  → read-only (GET allowed, PATCH rejected with 403)

version_number = MAX(version_number) for this story
  → editable (PATCH allowed)
```

### MagicLinkToken — Validity

```
used_at IS NULL AND expires_at > now()  → valid (can authenticate)
used_at IS NOT NULL                      → used (rejected)
expires_at <= now()                      → expired (rejected)
```

## Relationships

```
User (1) ──────────── (many) Story
Story (1) ─────────── (many) StoryVersion
User (1) ──────────── (many) MagicLinkToken  [by email, not FK]
```

## Migration Strategy (sessionStorage → DB on login)

1. After JWT cookie is set, frontend checks sessionStorage for story content
2. If any field is non-empty:
   - POST /api/stories with title = "My first story" (new account) or "Imported story" (existing account)
   - POST /api/stories/:id/versions with all field values from sessionStorage
3. Clear sessionStorage
4. Redirect to app with new story open
